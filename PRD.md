# GameInsight - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
GameInsight — 游戏玩家反馈智能分析系统

### 1.2 目标用户
游戏策划、游戏设计师、产品经理

### 1.3 核心价值主张
通过 AI 自动收集、分析、提纯分散在中文社交媒体上的玩家反馈，帮助设计团队快速洞察真实诉求，降低信息噪音，实现从反馈到迭代的闭环。

---

## 2. 功能模块

### 2.1 数据采集模块 (Data Collector)

**目标平台（MVP 阶段优先 5 个）：**
- Bilibili（视频评论）
- 微博（话题/超话/评论）
- TapTap（论坛帖子/评价）
- 小红书（笔记/评论）
- 抖音（视频评论）

**扩展平台（后续迭代）：**
- 百度贴吧
- 微信公众号/文章评论

**功能要求：**
- [ ] 关键词过滤：仅抓取与指定游戏（如：英雄联盟）相关的内容
- [ ] 增量更新：每日自动爬取新增内容，避免重复抓取
- [ ] 遵守 robots.txt，仅爬取公开可见内容
- [ ] 爬取频率限制：每个平台请求间隔 > 5 秒
- [ ] 数据匿名化：存储时使用 hash 代替用户真实 ID

---

### 2.2 AI 分析引擎 (Insight Engine)

**情感分析**
- [ ] 区分负面情绪（纯谩骂）vs 建设性批评
- [ ] 具备从情绪化内容中提炼潜在诉求的能力

**诉求提取**
- [ ] 从长文本中提取具体的设计建议、痛点描述
- [ ] 生成简洁的诉求摘要

**玩家身份识别**
- [ ] KOL / 大V（粉丝数 > 1万，或认证账号）
- [ ] 游戏行业从业者（简介含：策划、设计师、制作人等关键词）
- [ ] 资深玩家（游戏时长/等级/成就高）
- [ ] 新手玩家（低等级，提出基础问题）

**自动分类**
- [ ] UI/UX
- [ ] 数值平衡
- [ ] 剧情/叙事
- [ ] BUG / 技术问题
- [ ] 新功能建议
- [ ] 运营/活动

**AI  provider**
- Moonshot API（已有配置）

---

### 2.3 设计师端 Dashboard (Designer View)

**整体风格**
- Apple Human Interface Guidelines
- 深空灰 `#1D1D1F` + 苹果蓝 `#007AFF`
- 圆角 12px、毛玻璃（backdrop-blur）、柔和阴影
- 字体：系统默认无衬线字体栈
- 动效：300ms ease-in-out

**布局结构**
- 左侧固定导航栏（窄边栏）
- 右侧主内容区
- 响应式：支持移动端适配

**周报视图**
- [ ] 卡片式布局，展示本周热点诉求 TOP 10
- [ ] 每个卡片显示：
  - 诉求摘要（1-2 行）
  - 来源平台图标/标签
  - 玩家身份标签（彩色 badge）
  - 情感分值（-1 ~ +1）

**卡片交互**
- [ ] 点击卡片 → 平滑过渡动画展开为详情弹窗
- [ ] 详情页显示：
  - 诉求提取原文
  - AI 分析摘要
  - 来源链接（可跳转）
  - 相似诉求聚合列表

**任务转化**
- [ ] 详情页内提供「转为待办」按钮
- [ ] 设计师可添加备注和优先级

**待办管理**
- [ ] 看板视图：待处理 / 设计中 / 已完成
- [ ] 拖拽或点击变更状态
- [ ] 标记为已解决后，系统自动生成更新日志草稿

---

### 2.4 周报生成模块 (Weekly Report)

**自动生成**
- [ ] 每周一早晨自动生成并推送上周总结

**报告内容**
- [ ] 数据概览：总收集数 / 有效诉求数 / 热门分类 TOP 3
- [ ] 重点诉求详情（3-5 条）
- [ ] 趋势对比：与上周数据的环比变化

**导出**
- [ ] 支持导出 Markdown
- [ ] 支持导出 PDF（便于发布到游戏公告）

## 3. 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Frontend  │────▶│   Backend   │────▶│   SQLite    │
│  React+TW   │◀────│ Node/Express│◀────│   DB        │
└─────────────┘     └─────────────┘     └─────────────┘
                           ▲
                           │
                    ┌─────────────┐
                    │   Crawler   │
                    │Python+Playw │
                    └─────────────┘
                           │
                    ┌─────────────┐
                    │  Platforms  │
                    │  微博/B站/..│
                    └─────────────┘
```

### 3.1 前端
- React 18
- Tailwind CSS
- React Router（如需要多页面）
- Lucide React（图标）

### 3.2 后端
- Node.js + Express
- better-sqlite3
- node-cron（定时任务）
- axios / fetch（调用 Moonshot API）

### 3.3 爬虫
- Python 3.11+
- Playwright（模拟浏览器行为）
- APScheduler（定时调度）

### 3.4 数据库设计（V1）

**feeds** — 原始抓取内容
- id, platform, url, content, author_hash, published_at, collected_at, game_keyword

**insights** — AI 分析结果
- id, feed_id, summary, sentiment_score, category, player_type, ai_analyzed_at

**todos** — 设计师待办
- id, insight_id, title, status (pending/designing/done), notes, priority, created_at, updated_at

**weekly_reports** — 周报
- id, week_start, week_end, summary_json, generated_at

## 4. 隐私与合规

- 仅爬取**公开可见**内容
- 存储时隐藏真实用户 ID（SHA256 hash）
- 爬取频率限制（> 5s / 请求）
- 提供「删除请求」入口（模拟玩家要求删除自己的反馈数据）
- 遵守各平台 robots.txt

---

## 5. MVP 范围

### 必须实现（求职作品展示用）
1. 支持 5 个平台的数据采集（B站、微博、TapTap、小红书、抖音）
2. 基础 AI 分析（情感过滤 + 诉求提取）
3. 周报卡片视图（TOP 10 卡片展示）
4. 待办状态流转（待处理 → 设计中 → 已完成）
5. 苹果风格的 UI 设计（重点展示前端审美）

### 后续迭代
- 百度贴吧、微信公众号支持
- 更精细的玩家画像
- 高级筛选与搜索
- 多项目管理（同时跟踪多款游戏）
- 邮件/企业微信推送

## 7. 用户流程

1. 设计师配置：输入游戏名称、关键词、目标平台
2. 系统每日自动爬取 → AI 分析 → 存入数据库
3. 周一早晨生成周报推送至设计师端
4. 设计师浏览卡片 → 点击展开详情 → 选择「转为待办」
5. 设计师在看板中管理迭代任务
6. 任务完成后标记 → 系统自动生成更新公告草稿
